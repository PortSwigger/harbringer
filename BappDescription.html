HARBringer is a Burp Suite extension developed by Abira Security specifically designed to import HTTP Archive (HAR) files into Burps site map. Unlike other similar extensions, it ensures all the original HTTP requests and responses of your HAR file are preserved; this is to say, it does not request each entry in the HAR file upon importation but retains the originality of the requests and responses. With HARBringer, you can analyze previously captured traffic without the necessity to recapture it through Burps proxy.

Features

  • Ability to import entire HAR files with a single click.
  • A sort-able table that presents HAR entries.
  • Importation options that allow you to either import individual entries or all entries at once.
  • Support for binary data and various encodings.
  • Preservation of request and response headers and bodies.
  • Direct addition of entries to Burps site map for further analysis.

Usage

  1. After integrating the extension, locate the HARBringer tab within Burp Suite.
  2. Click Browse to select a desired HAR file.
  3. Choose Load HAR to load the entries from the file into the table displayed.
  4. To import a single entry, select it in the table and click Send to Site Map.
  5. To import all entries, simply click Import All to Site Map.
  6. The imported entries will then appear in Burps site map, ready for analysis like normal traffic.

Supported HAR Formats

HARbringer supports HAR files generated by several tools. While some have been thoroughly tested, others are untested.

Tested Tools:

  • Chrome DevTools
  • Firefox Developer Tools
  • Edge DevTools
  • Safari Web Inspector

Untested Tools:

  • Fiddler
  • Charles Proxy
  • ZAP Proxy

Moreover, HARBringer has special handling for binary data, such as images, PDFs, and other non-text content. It can process Base64-encoded content, binary content with various MIME types, and content with different character encodings.

If you encounter issues while importing certain entries, check the extension output in the Output tab of the Extensions section. Look for specific error messages that might indicate the problem. For binary data issues, ensure the HAR file correctly specifies the MIME types.
